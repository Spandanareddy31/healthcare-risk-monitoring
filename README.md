# Real-Time Healthcare Risk Monitoring Using Distributed Data Pipelines

## Course
DSCI 6007 – Distributed & Scalable Data Engineering  
Midterm Project – Phase 1

## Team Name
VitalStream Analytics

## Project Overview
Hospitals generate continuous, high-velocity patient vitals data from ICU monitors and medical devices. Delays in detecting abnormal vitals can lead to critical consequences.

This project designs a scalable distributed data pipeline to ingest, process, store, and serve healthcare risk insights in real-time.

---

## Engineering Objective
Design a distributed pipeline that:
1. Ingests streaming vitals data
2. Processes and enriches it at scale
3. Stores it for fast querying
4. Serves risk insights to end users

---

## Four Pillars of Data Engineering
1. Data Ingestion – Simulated streaming JSON events
2. Data Processing – Spark Structured Streaming
3. Data Storage – Parquet storage
4. Data Consumption – Streamlit dashboard
